Created 2026-04-30. User wanted parallel GW2 settings (Nexus keybinds, Mystic Clicker resolution config, GFXSettings, addon UI positions) per stream client (Deck vs Apple TV) without manually re-editing GW2 every switch. Two installs share the heavy game content (`Gw2.dat`, `bin64/`) and diverge on addons + Wine prefix.

## Layout

| Path | Type | Purpose |
|---|---|---|
| `~/.local/share/Steam/steamapps/common/Guild Wars 2/` | Steam-managed | **Deck profile** install — Steam app id 1284210 |
| `~/Games/gw2-appletv/` | Manual | **Apple TV profile** install — non-Steam shortcut |
| `~/Games/gw2-appletv/Gw2.dat` | symlink → Steam install | 81 GB content blob — patches flow through |
| `~/Games/gw2-appletv/bin64/` | symlink → Steam install | Steam-patched binaries flow through |
| `~/Games/gw2-appletv/Gw2-64.exe` | copy | 43 MB launcher; sync-gw2-exe.sh re-copies after Steam patches |
| `~/Games/gw2-appletv/addons/` | independent copy | Diverges per profile (Nexus, MysticClicker, ArcDPS, etc.) |
| `~/.local/share/Steam/steamapps/compatdata/1284210/pfx/` | Wine prefix A | Deck profile GFXSettings + user.reg |
| `~/.local/share/Steam/steamapps/compatdata/<new-appid>/pfx/` | Wine prefix B | Apple TV profile GFXSettings + user.reg (auto-created on first launch of non-Steam shortcut) |

Net unique disk cost: ~200 MB (43 MB exe + ~150 MB addons). Gw2.dat (81 GB) and bin64 (204 MB) are single-source via symlink.

## How patches flow

- **Gw2.dat content patches** (most ArenaNet content patches): Steam writes to `Gw2.dat` in the Steam install. Apple TV profile's symlink resolves at access time, so it reads the new content automatically. **No action needed.**
- **bin64/ updates** (rare DXGI / D3D11 / utility binaries): symlink, flows through. **No action needed.**
- **Gw2-64.exe launcher patches** (occasional): the launcher is a copy, so the Apple TV install stays on the old version until manually re-copied. Run `scripts/sync-gw2-exe.sh` (deployed to `/var/home/Og/scripts/sync-gw2-exe.sh`) to detect and re-copy when needed. Idempotent.

## Steam Input controller mapping

Each non-Steam shortcut gets a unique appid auto-generated by Steam (large 32-bit number). Add an entry for the new appid → desired controller template in:

`~/.local/share/Steam/userdata/64793831/config/configset_controller_neptune.vdf`

Mirror to `configs/steam-deck/configset_controller_neptune.vdf` in this repo.

## Sunshine flow

The existing **"GW2 - SteamOS"** Sunshine app handles both clients. Moonlight from either Deck or Apple TV launches Big Picture; user picks the correct profile from the Steam library:

- **Deck stream** → pick **"Guild Wars 2"** in Big Picture (Steam app, Deck profile)
- **Apple TV stream** → pick **"Guild Wars 2 (Apple TV)"** in Big Picture (non-Steam shortcut, Apple TV profile)

The prep-cmd (`sunshine-set-resolution.sh`) auto-swaps Wine LogPixels per client width — that runs against whichever Wine prefix the chosen Steam app uses, so DPI is correct for either profile. **No Sunshine app changes were required for the dual-install setup.**

## Reversibility

If the dual install causes problems:

1. Delete `~/Games/gw2-appletv/` (no harm to the Steam install — symlinks are inert)
2. Remove the non-Steam shortcut from Steam library
3. Revert `configset_controller_neptune.vdf` from repo backup

The Steam install at `~/.local/share/Steam/steamapps/common/Guild Wars 2/` is never modified by this setup.

## Critical rules for future sessions

- **Never write to the Apple TV install with `cp -r` from the Steam install** — that would clobber the Apple TV profile's diverged Nexus configs, Mystic Clicker positions, etc. Only `Gw2-64.exe` should be re-copied from Steam, and only via `sync-gw2-exe.sh`.
- **Never replace the symlinks with copies** — that would break patch flow for `Gw2.dat` and `bin64/`, doubling disk use and creating a sync chore.
- **Symlinks must stay relative-resolvable**: if the Steam install is moved (different library), update both symlinks: `~/Games/gw2-appletv/Gw2.dat` and `~/Games/gw2-appletv/bin64`.
- **Account login is single-instance**: ArenaNet enforces one login per account; second launch boots the first. User confirmed never running both simultaneously.

## Backup strategy

The two profiles' Nexus configs should be backed up to the repo separately so each client's settings are versioned:

- `configs/gw2-keybinds/deck/` — Nexus configs from `~/.local/share/Steam/steamapps/common/Guild Wars 2/addons/Nexus/`
- `configs/gw2-keybinds/apple-tv/` — Nexus configs from `~/Games/gw2-appletv/addons/Nexus/`

Files to back up per profile: `InputBinds.json`, `GameBinds.xml`, `AddonConfig.json`, `Settings.json`. Mystic Clicker `.cfg` files live in `addons/MysticClicker/` per profile.
